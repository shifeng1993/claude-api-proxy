# 本地安装部署

本文档说明如何在本机或服务器部署 Claude API Proxy。所有地址、账号、数据库、LDAP、邮件、上游模型服务都应通过 `.env` 或控制台动态配置，不在代码和文档中写死真实组织信息。

## 一、准备环境

运行依赖：

- Node.js 18 或更高版本，推荐 Node.js 22 LTS。
- MySQL 8 或兼容数据库。
- 可选：PM2，用于生产进程守护。
- 可选：Nginx，用于 HTTPS 和 WebSocket 反向代理。

部署包至少应包含：

```text
package.json
package-lock.json
src/
public/
ecosystem.config.cjs
.env.example
```

## 二、本机快速启动

1. 安装依赖：

```bash
npm install
```

2. 准备环境变量：

```bash
cp .env.example .env
```

Windows PowerShell：

```powershell
Copy-Item .env.example .env
```

3. 修改 `.env`。本地账号模式的最小配置：

```env
PORT=3080
HOST=0.0.0.0
JWT_SECRET=<a-long-random-secret>

DB_DIALECT=mysql
DB_HOST=127.0.0.1
DB_PORT=3306
DB_USER=root
DB_PASSWORD=<db-password>
DB_NAME=claude_api_proxy

LOCAL_ADMIN_USER=admin
LOCAL_ADMIN_PASSWORD=<at-least-8-chars>
```

LDAP 模式还需要：

```env
LDAP_SERVER=ldap://ldap.example.com:389
LDAP_BIND_DN=<ldap-bind-account>
LDAP_BIND_PASSWORD=<ldap-bind-password>
LDAP_BASE_DN=DC=example,DC=com
LDAP_FILTER=(sAMAccountName={userNo})
```

启动时服务会探测 LDAP 是否可达。LDAP 可达时登录页显示“当前是LDAP认证模式”；LDAP 不可达或未配置时显示“当前是本地账号认证模式”。

4. 启动服务：

```bash
npm start
```

开发模式：

```bash
npm run dev
```

5. 打开控制台：

```text
http://127.0.0.1:3080/admin
```

控制台使用 hash 路由，例如 `#/dashboard/overview`、`#/dashboard/relay`、`#/stats/relay/users`。刷新页面不会回到默认页。

## 三、首次初始化

1. 使用 `LOCAL_ADMIN_USER` 和 `LOCAL_ADMIN_PASSWORD` 登录，或在 LDAP 模式下使用域账号登录。
2. 在“服务控制台”查看当前用户统一 API Key。
3. 按需启用 Relay、CodeBuddy、Copilot。
4. Relay：添加上游，选择协议类型，配置 Base URL、API Key、代理、跳过 TLS 证书验证和模型映射。
5. CodeBuddy：新增凭证，选择默认上游或自定义上游。
6. Copilot：新增凭据，按设备授权流程连接 GitHub。
7. 管理员可继续配置用户管理、使用统计和问题反馈。

## 四、Claude Code 配置

登录控制台后，复制当前用户的统一 API Key。推荐使用 `ANTHROPIC_AUTH_TOKEN`。

Relay 示例：

```json
{
  "env": {
    "ANTHROPIC_AUTH_TOKEN": "sk-xxx",
    "ANTHROPIC_BASE_URL": "http://127.0.0.1:3080/relay/anthropic",
    "NO_PROXY": "localhost,127.0.0.1,::1"
  }
}
```

CodeBuddy 示例：

```json
{
  "env": {
    "ANTHROPIC_AUTH_TOKEN": "sk-xxx",
    "ANTHROPIC_BASE_URL": "http://127.0.0.1:3080/codebuddy/anthropic",
    "NO_PROXY": "localhost,127.0.0.1,::1"
  }
}
```

Copilot 示例：

```json
{
  "env": {
    "ANTHROPIC_AUTH_TOKEN": "sk-xxx",
    "ANTHROPIC_BASE_URL": "http://127.0.0.1:3080/copilot/anthropic",
    "NO_PROXY": "localhost,127.0.0.1,::1"
  }
}
```

旧版鉴权兼容逻辑仍保留，新配置建议统一使用 `ANTHROPIC_AUTH_TOKEN`。

## 五、API 入口

所有入口都使用统一 API Key 鉴权。

### Relay

```text
POST /relay/anthropic/v1/messages
POST /relay/anthropic/v1/messages/count_tokens
GET  /relay/anthropic/v1/models
POST /relay/v1/chat/completions
POST /relay/v1/responses
POST /relay/v1/responses/compact
GET  /relay/v1/models
WS   /relay/v1/responses
```

Relay 上游支持 `openai`、`responses`、`responses_ws`、`anthropic` 四种协议。Anthropic、OpenAI Chat、Responses HTTP、Responses WS 客户端入口可以交叉调用这些上游。流式路径会尽量边收边转，减少多跳场景延迟。

Relay 会维护短期内存会话状态，用于 Responses/Responses WebSocket 增量请求转到 Chat 或 Anthropic 上游时恢复完整上下文。状态按租户和会话隔离，当前推荐单实例部署；默认保留 24 小时并定期清理，服务重启后内存状态会丢失，未来重新扩容时再引入共享存储。

### CodeBuddy

```text
POST /codebuddy/anthropic/v1/messages
POST /codebuddy/anthropic/v1/messages/count_tokens
GET  /codebuddy/anthropic/v1/models
POST /codebuddy/v1/chat/completions
POST /codebuddy/v1/responses
POST /codebuddy/v1/responses/compact
GET  /codebuddy/v1/models
WS   /codebuddy/v1/responses
```

额外上游地址通过环境变量配置：

```env
CODEBUDDY_EXTRA_BASE_URLS=https://codebuddy.example.com,https://ai-coding.example.com
CODEBUDDY_CUSTOM_SITE_LABELS={"ai-coding.example.com":"自定义站"}
```

模型列表覆盖：

```env
CODEBUDDY_MODEL_OVERRIDES={"ai-coding.example.com":[{"id":"custom:glm51","name":"GLM-5.1","tools":true,"vision":false}]}
```

### Copilot

```text
POST /copilot/anthropic/v1/messages
POST /copilot/anthropic/v1/messages/count_tokens
GET  /copilot/anthropic/v1/models
POST /copilot/v1/chat/completions
POST /copilot/v1/responses
POST /copilot/v1/responses/compact
GET  /copilot/v1/models
WS   /copilot/v1/responses
```

## 七、服务器部署

### 单实例 PM2

```bash
npm install
pm2 start ecosystem.config.cjs
pm2 save
pm2 startup
pm2 list
```

常用命令：

```bash
pm2 logs claude-api-proxy
pm2 restart claude-api-proxy
pm2 stop claude-api-proxy
```

### Nginx 示例

Responses WebSocket 与 HTTP 共用 `/v1/responses` 路径，反向代理必须保留 Upgrade 头。

```nginx
upstream claude_api_proxy {
    server 127.0.0.1:3081;
    server 127.0.0.1:3082;
    server 127.0.0.1:3083;
    server 127.0.0.1:3084;
}

server {
    listen 3080;
    server_name _;

    client_max_body_size 20m;
    proxy_read_timeout 3600s;
    proxy_send_timeout 3600s;

    location / {
        proxy_pass http://claude_api_proxy;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
    }
}
```

### Cloudflare 注意事项

通过 Cloudflare 代理（橙色云朵）访问时，**Cloudflare WAF 会拦截包含 `/admin` 的请求**，返回 `Sorry, you have been blocked` 页面。这是因为 Cloudflare 的托管规则会将 `/admin` 视为敏感后台路径。

**解决方案**：本项目的管理面板路径已从 `/admin` 改为 `/dashboard`，hash 路由从 `#/dashboard` 改为 `#/console`，以绕过 Cloudflare 规则。如需映射到其他路径，修改以下文件中的路径前缀：

- `src/server.js`：路由匹配
- `src/routes/dashboard-frontend.js`：子路由
- `src/templates/admin.html`：前端 API 调用和 hash 路由

或通过 Cloudflare 控制台 Security → WAF → Custom Rules 添加例外规则允许 `/admin` 路径。

| `PORT` | 服务监听端口，默认 `3080` |
| `HOST` | 服务监听地址，默认 `0.0.0.0` |

| `DB_PORT` | 数据库端口，默认 `3306` |
| `DB_USER` | 数据库用户，默认 `root` |
| `DB_PASSWORD` | 数据库密码，默认空 |
| `DB_NAME` | 数据库名，默认 `claude_api_proxy` |
| `LDAP_SERVER` | LDAP 服务地址 |
| `LDAP_BIND_DN` | LDAP 绑定账号 |
| `LDAP_BIND_PASSWORD` | LDAP 绑定密码 |
| `LDAP_BASE_DN` | LDAP 搜索基准 DN |
| `LDAP_FILTER` | LDAP 用户过滤器，默认 `(sAMAccountName={userNo})` |
| `LDAP_PROBE_TIMEOUT_MS` | LDAP 启动探测超时，默认 `3000` |
| `LOCAL_ADMIN_USER` | 本地账号模式初始 superadmin |
| `LOCAL_ADMIN_PASSWORD` | 本地账号模式初始 superadmin 密码 |
| `CODEBUDDY_REGION` | CodeBuddy 默认区域，`cn` 或 `intl` |

| `CODEBUDDY_MODEL_OVERRIDES` | 自定义上游模型列表覆盖，模型项支持 `id`、`name`、`tools`、`vision` |
| `RELAY_CONVERSATION_STATE_TTL_MS` | Relay 内存会话状态保留时间，默认 `86400000` |

## 九、常见问题

### 启动后无法登录

- 看启动日志确认当前是 LDAP 模式还是本地账号模式。
- LDAP 模式下检查 LDAP 服务、绑定账号、搜索基准 DN 和过滤器。
- 本地账号模式下确认配置了 `LOCAL_ADMIN_USER` 和长度不少于 8 的 `LOCAL_ADMIN_PASSWORD`。

### API 返回 401

- 确认客户端使用的是当前用户的 `sk-...` API Key。
- 推荐使用 `ANTHROPIC_AUTH_TOKEN`，不要把登录密码当作 API Key。
- 确认对应服务已在控制台启用并完成凭证或上游配置。

### Relay 上游模型名不匹配

在 Relay 上游配置中添加模型映射，把客户端模型名映射到上游真实模型名。未配置映射时，模型名会原样透传。

### WebSocket 连接失败

- 确认反向代理保留 `Upgrade` 和 `Connection` 头。
- 确认客户端使用 `ws://` 或 `wss://` 访问 `/relay/v1/responses`、`/codebuddy/v1/responses` 或 `/copilot/v1/responses`。
- 如果上游是自签证书，可在上游配置中填写代理后勾选“跳过 TLS 证书验证”，或直连时直接勾选该选项。

## 十、升级建议

1. 备份 `.env` 和数据库。
2. 替换新版本代码。
3. 执行 `npm install`。
4. 对照 `.env.example` 补齐新增变量。
5. 重启服务。
6. 检查 `/admin`、`/health`、Relay/CodeBuddy/Copilot 关键 API 和 WebSocket 入口。
