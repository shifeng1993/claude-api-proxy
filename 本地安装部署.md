# 本地安装部署

本文档说明如何在本机或服务器部署 Claude API Proxy。所有真实地址、账号、数据库、LDAP、邮件和上游模型服务都应通过 `.env` 或控制台配置，不要写死在代码或文档里。

## 一、准备环境

运行依赖：

- Node.js 18 或更高版本，推荐 Node.js 22 LTS。
- MySQL 8 或兼容数据库。
- 可选：PM2，用于生产进程守护。
- 可选：Nginx、Caddy 或其他反向代理，用于 HTTPS 和 WebSocket。

部署包至少应包含：

```text
package.json
package-lock.json
src/
public/
scripts/
ecosystem.config.cjs
.env.example
README.md
本地安装部署.md
```

## 二、本机快速启动

1. 安装依赖：

```bash
npm install
```

2. 准备环境变量：

```bash
cp .env.example .env
```

Windows PowerShell：

```powershell
Copy-Item .env.example .env
```

3. 修改 `.env`。本地账号模式的最小配置：

```env
PORT=3080
HOST=0.0.0.0
JWT_SECRET=<a-long-random-secret>

DB_DIALECT=mysql
DB_HOST=127.0.0.1
DB_PORT=3306
DB_USER=<your-db-user>
DB_PASSWORD=<your-db-password>
DB_NAME=claude_api_proxy

LOCAL_ADMIN_USER=admin
LOCAL_ADMIN_PASSWORD=<at-least-8-chars>
```

LDAP 模式还需要：

```env
LDAP_SERVER=ldap://ldap.example.com:389
LDAP_BIND_DN=<ldap-bind-account>
LDAP_BIND_PASSWORD=<ldap-bind-password>
LDAP_BASE_DN=DC=example,DC=com
LDAP_FILTER=(sAMAccountName={userNo})
```

只有 LDAP 必要变量都设置且 LDAP 服务可达时，启动时才会启用 LDAP 登录；否则自动回退到本地账号模式。

4. 启动服务：

```bash
npm start
```

开发模式：

```bash
npm run dev
```

5. 打开控制台：

```text
http://127.0.0.1:3080/dashboard
```

控制台使用 hash 路由，例如 `#/relay`、`#/codebuddy`、`#/stats/relay/monthly`、`#/feedback`、`#/users`。根路径 `/` 会根据登录状态跳转到 `/login` 或 `/dashboard#/relay`。

## 三、首次初始化

1. 使用 `LOCAL_ADMIN_USER` 和 `LOCAL_ADMIN_PASSWORD` 登录，或在 LDAP 模式下使用域账号登录。
2. 在控制台查看当前用户的统一 API Key。
3. 按需启用 Relay、CodeBuddy。
4. Relay：添加上游，选择协议类型，配置 Base URL、API Key、代理、跳过 TLS 证书验证、模型列表和模型映射。
5. CodeBuddy：新增凭证，选择默认上游或自定义上游，也可以通过 OAuth 授权流程添加凭证。
6. 管理员可继续配置用户管理、使用统计和问题反馈。

## 四、Claude Code 配置

登录控制台后，复制当前用户的统一 API Key。推荐统一使用 `ANTHROPIC_AUTH_TOKEN`。

Relay 示例：

```json
{
  "env": {
    "ANTHROPIC_AUTH_TOKEN": "sk-xxx",
    "ANTHROPIC_BASE_URL": "http://127.0.0.1:3080/relay/anthropic",
    "NO_PROXY": "localhost,127.0.0.1,::1"
  }
}
```

CodeBuddy 示例：

```json
{
  "env": {
    "ANTHROPIC_AUTH_TOKEN": "sk-xxx",
    "ANTHROPIC_BASE_URL": "http://127.0.0.1:3080/codebuddy/anthropic",
    "NO_PROXY": "localhost,127.0.0.1,::1"
  }
}
```

新版配置建议只使用 `ANTHROPIC_AUTH_TOKEN`。旧版鉴权兼容逻辑仍保留，但不要把登录密码当作 API Key。

## 五、API 入口

所有入口都使用统一 API Key 鉴权。只有当前用户已启用对应服务时，请求才会被放行。

### Relay

```text
POST /relay/anthropic/v1/messages
POST /relay/anthropic/v1/messages/count_tokens
GET  /relay/anthropic/v1/models

POST /relay/v1/chat/completions
POST /relay/v1/responses
POST /relay/v1/responses/compact
GET  /relay/v1/models
GET  /relay/v1/diagnostics

POST /relay/v1/telemetry/report
GET  /relay/v1/telemetry/source-check

WS   /relay/v1/responses
```

Relay 上游支持 `openai`、`responses`、`responses_ws`、`anthropic` 四种协议。Anthropic、OpenAI Chat、Responses HTTP、Responses WS 客户端入口可以交叉调用这些上游。流式路径会尽量边收边转，减少多跳场景延迟。

Relay 会维护短期内存会话状态，用于 Responses/Responses WebSocket 增量请求转到 Chat 或 Anthropic 上游时恢复完整上下文。状态按租户和会话隔离，当前推荐单实例部署；默认保留 24 小时并定期清理，服务重启后内存状态会丢失。

### CodeBuddy

```text
POST /codebuddy/anthropic/v1/messages
POST /codebuddy/anthropic/v1/messages/count_tokens
GET  /codebuddy/anthropic/v1/models

POST /codebuddy/v1/chat/completions
POST /codebuddy/v1/responses
POST /codebuddy/v1/responses/compact
GET  /codebuddy/v1/models

GET/POST /codebuddy/v1/credentials
POST     /codebuddy/v1/telemetry/report
GET      /codebuddy/v1/telemetry/source-check

WS       /codebuddy/v1/responses
```

额外上游地址通过环境变量配置：

```env
CODEBUDDY_EXTRA_BASE_URLS=https://codebuddy.example.com,https://ai-coding.example.com
CODEBUDDY_CUSTOM_SITE_LABELS={"ai-coding.example.com":"自定义站"}
```

模型列表覆盖：

```env
CODEBUDDY_MODEL_OVERRIDES={"ai-coding.example.com":[{"id":"custom:glm51","name":"GLM-5.1","tools":true,"vision":false}]}
```

### 兼容路径

服务会在路由前做路径规范化，方便兼容旧客户端或网关前缀：

| 输入路径 | 实际路由 |
| --- | --- |
| `/api/login`、`/api/logout`、`/api/dashboard/...` | `/login`、`/logout`、`/dashboard/...` |
| `/api/usage...` | `/stats/api...` |
| `/api/stats...` | `/stats/api...` |
| `/coding/relay/...`、`/api/coding/relay/...` | `/relay/...` |
| `/coding/codebuddy/...`、`/api/coding/codebuddy/...` | `/codebuddy/...` |

健康检查：

```text
GET /health
```

## 六、服务器部署

### 单实例 PM2

```bash
npm install
pm2 start ecosystem.config.cjs
pm2 save
pm2 startup
pm2 list
```

`ecosystem.config.cjs` 当前使用应用名 `ClaudeApiProxy`，并设置了：

```text
NODE_OPTIONS=--max-old-space-size=768
max_memory_restart=1000M
```

常用命令：

```bash
pm2 logs ClaudeApiProxy
pm2 restart ClaudeApiProxy
pm2 stop ClaudeApiProxy
```

当前推荐单实例部署。Relay conversation state 存在进程内，单实例能保证 Responses/WS 增量请求在协议切换时稳定命中上下文。多实例部署前需要先考虑会话状态共享或会话粘滞。

### Nginx 示例

Responses WebSocket 与 HTTP 共用 `/v1/responses` 路径，反向代理必须保留 Upgrade 头。以下示例让 Nginx 监听 80，并转发到本机 3080：

```nginx
upstream claude_api_proxy {
    server 127.0.0.1:3080;
}

server {
    listen 80;
    server_name _;

    client_max_body_size 20m;
    proxy_read_timeout 3600s;
    proxy_send_timeout 3600s;

    location / {
        proxy_pass http://claude_api_proxy;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
    }
}
```

如果使用 HTTPS，请在反向代理层配置证书，并确认 WebSocket 客户端使用 `wss://`。

### Cloudflare 注意事项

通过 Cloudflare 代理访问时，建议保留默认控制台路径 `/dashboard`，并确认 WAF 规则不会拦截控制台和 API 路径。Responses WebSocket 需要 Cloudflare 侧允许 WebSocket，并保持较长的读写超时。

如需映射到其他路径，需要同步修改：

- `src/server.js`：路由匹配和路径规范化。
- `src/routes/dashboard-frontend.js`：控制台页面和 API 子路由。
- `src/templates/admin.html`：前端 API 调用和 hash 路由。

## 九、常见问题

### 启动后无法登录

- 看启动日志确认当前是 LDAP 模式还是本地账号模式。
- LDAP 模式下检查 LDAP 服务、绑定账号、搜索基准 DN 和过滤器。
- 本地账号模式下确认配置了 `LOCAL_ADMIN_USER` 和长度不少于 8 的 `LOCAL_ADMIN_PASSWORD`。
- 确认 `JWT_SECRET` 已配置，且浏览器允许保存登录 Cookie。

### API 返回 401 或 503

- 401 通常表示 API Key 缺失或无效，确认客户端使用的是当前用户的 `sk-...` API Key。
- 503 通常表示对应服务未启用，确认 Relay 或 CodeBuddy 已在控制台为该用户启用。
- 不要把控制台登录密码当作 API Key。

### Relay 上游模型名不匹配

在 Relay 上游配置中添加模型映射，把客户端模型名映射到上游真实模型名。未配置映射时，模型名会原样透传；`model_auto` 未关闭时会对部分常见模型名做兜底映射。

### WebSocket 连接失败

- 确认反向代理保留 `Upgrade` 和 `Connection` 头。
- 确认客户端使用 `ws://` 或 `wss://` 访问 `/relay/v1/responses` 或 `/codebuddy/v1/responses`。
- 如果上游是自签证书，可在上游配置中勾选“跳过 TLS 证书验证”。

### 长任务或多轮续写上下文丢失

- 当前推荐单实例部署，避免请求落到不同进程导致内存会话状态不一致。
- 检查 `RELAY_CONVERSATION_STATE_TTL_MS` 是否过短。
- 小内存环境可以配置会话数量和字节预算，但预算过紧会更早驱逐旧会话。

## 十、升级建议

1. 备份 `.env` 和数据库。
2. 替换新版本代码。
3. 执行 `npm install`。
4. 对照 `.env.example` 补齐新增变量。
5. 重启服务。
6. 检查 `/dashboard`、`/health`、Relay/CodeBuddy 关键 HTTP API 和 WebSocket 入口。
7. 运行 `npm.cmd test` 或 `npm test` 验证当前环境下的基础测试。
